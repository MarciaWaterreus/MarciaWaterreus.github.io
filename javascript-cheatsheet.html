let Let keyword This keyword signals that the variable can be reassigned a different value or changed later on. It is possible to declare a variable without assigning the variable a value. In such a case, the variable will be automatically initialized with a value of undefined.
const Constant keyword This variable cannot be reassigned because it is constant. If you try to reassign a const variable, you'll get a TypeError. Constant variables must be assigned a value when declared. If you try to declare a const variable without a value, you'll get a SyntaxError.
var Variable keyword There are a few general rules for naming variables:
  • They cannot start with numbers.
  • They are case sensitive, so myName and myname would be different variables. It is bad practice to create two variables that have the same name using different cases.
  • Their names cannot be the same as keywords. For a comprehensive list of keywords check out MDN's keyword documentation.

name = (param1, param2) => { // code to be executed; }		 Function or fat arrow syntax Arrow functions remove the need to type out the keyword function every time you need to create a function. Instead, you first include the parameters inside the () and then add an arrow => that points to the function body surrounded in {}.
Function Declarations vs. Function Expressions
Zero parameters:
const functionName = ( ) => { };

One parameter:
const functionName = paramOne => { };

Two or more parameters:
const functionName = (paramOne, paramTwo) => { };

Single-line block:
const sumNumbers = number => number + number;

Multi-line block:
const sumNumbers = number => {
const sum = number + number;
return sum;
}; 		Concise Body Arrow Functions JavaScript also provides several ways to refactor arrow function syntax. The most condensed form of the function is known as concise body:
Functions that take only a single parameter do not need that parameter to be enclosed in parentheses. However, if a function takes zero or multiple parameters, parentheses are required.
A function body composed of a single-line block does not need curly braces. Without the curly braces, whatever that line evaluates will be automatically returned. The contents of the block should immediately follow the arrow => and the return keyword can be removed. This is referred to as implicit return.
